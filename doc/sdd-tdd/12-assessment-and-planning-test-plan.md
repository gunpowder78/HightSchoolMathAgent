# 学情评估与个性化补习计划工具测试方案

## 1. 测试目标

本方案服务于 `11-assessment-and-planning-tool-spec.md`，验证该工具是否真正做到：

- 学情评估准确；
- 计划生成可解释；
- 风险识别不失真；
- 调参与熔断逻辑可控；
- 输出符合研究报告中的防御型与双轨重构原则。

## 2. TDD 原则

本工具开发必须遵循以下节奏：

1. 先为评分规则写失败测试。
2. 再为计划生成器写失败测试。
3. 最后实现最小通过代码。
4. 任何规则调整都必须先更新测试样例。

## 3. 测试分层

### 3.1 单元测试

覆盖对象：

- 学情问卷解析器；
- 模块能力评分器；
- 心理风险评分器；
- 白名单/黑名单规则引擎；
- 主线选择器；
- 计划生成器；
- 熔断修复触发器。

### 3.2 集成测试

覆盖对象：

- 评估输入到画像输出流程；
- 画像输入到计划生成流程；
- 复评输入到计划调参流程；
- 计划输出到家长执行视图流程。

### 3.3 端到端测试

覆盖对象：

- 首次建档后生成首个 2 周计划；
- 一周训练后进行复评并调参；
- 连续失败后进入熔断修复模式。

### 3.4 人工抽检

覆盖对象：

- 计划话术是否打击学生；
- 建议是否符合艺术生认知优势；
- 黑名单限制是否足够坚决；
- 家长说明是否可执行。

## 4. 核心测试主题

### 4.1 学情采集完整性

验证点：

- 是否收齐目标分数、可用时长、薄弱模块、学校环境等关键字段。
- 缺失信息时是否提示补充，而不是生成伪完整画像。

### 4.2 防御底盘测算正确性

验证点：

- 是否把基础题、第一问和动态防区区分开。
- 是否避免简单按总分线性映射。

### 4.3 认知风格识别

验证点：

- 视觉型学生是否被识别为优先适配图像化路径。
- 对纯代数耐受度低的学生是否避免错误过桥方案。

### 4.4 风险识别

验证点：

- 是否能识别习得性无助风险。
- 是否能识别学校与家庭策略冲突。
- 是否能识别黑名单污染和时间预算不足。

### 4.5 计划个性化程度

验证点：

- 两个不同学生不能生成几乎相同的补习计划。
- 主线 A/B 顺序必须随画像变化。

### 4.6 熔断修复触发

验证点：

- 连续两次在同一认知节点失败时，是否暂停推进。
- 是否输出对应的降维修复方案。

## 5. 样例测试用例

| 用例 ID | 覆盖需求 | 场景 | 期望结果 |
| :---- | :---- | :---- | :---- |
| `APT-TC-01` | `APT-FR-01` `APT-FR-02` | 学生首次建档但缺少目标分数 | 系统提示补充，不生成正式计划 |
| `APT-TC-02` | `APT-FR-04` | 学生基础题正确率高，但大题后两问全失分 | 评估为“防御底盘尚可，压轴不可依赖” |

| `APT-TC-12` | `APT-FR-14` | 家长查看执行面板 | 输出今日重点、禁止项、鼓励话术 |

## 6. 测试数据集建议

建议建立：

- `eval/assessment_profiles.json`
  - 不同学生画像样本。
- `eval/planning_personas.json`
  - 视觉型、焦虑型、基础薄弱型等人物卡。
- `eval/risk_events.json`
  - 重点班压力、家庭冲突、黑名单污染样例。
- `eval/repair_protocols.json`
  - Bug 触发与修复建议样例。

## 7. 验收门禁

### 首期发布前必须满足

- 所有 `APT-FR` 对应至少一条测试用例。
- 至少 `80%` 的评分逻辑具备单元测试。
- 至少完成一轮人工抽检，确认话术不过度打击学生。
- 至少有两类完全不同画像能生成显著不同的计划。

### 二阶段强化前必须满足

- 计划调参与历史版本回溯可用。
- 熔断修复协议可追溯到具体失败节点。

## 8. 回归策略

- 修改评分规则：
  - 跑单元测试 + 画像样例回归。
- 修改计划生成逻辑：
  - 跑集成测试 + 人工抽检。
- 修改风险模型：
  - 跑风险样例集 + 计划差异对比测试。
- 修改家长报告模版：
  - 跑人工可执行性抽检。
