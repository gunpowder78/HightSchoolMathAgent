# M1 本地 MVP Backlog

## 1. 目的

本文件将 `M1 本地 MVP` 范围内的需求拆解为可开发 issue，供后续迭代按优先级推进。

## 2. Issue 列表

### `ISSUE-M1-01` 项目初始化与运行入口

- 对应需求：
  - `NFR-03`
  - `NFR-04`
- 目标：
  - 建立项目目录结构、配置入口、依赖文件和本地启动方式。
- 交付物：
  - `app/main.py`
  - `app/config.py`
  - `requirements.txt`
  - 根目录 `README.md`
- 验收标准：
  - 本地能成功启动应用骨架。

### `ISSUE-M1-02` SQLite 持久化底座

- 对应需求：
  - `FR-12`
  - `FR-13`
  - `NFR-04`
- 目标：
  - 提供本地会话和错题记录的最小持久化能力。
- 交付物：
  - `app/persistence/sqlite_store.py`
- 验收标准：
  - 可创建会话并写入/读取轮次记录。

### `ISSUE-M1-03` 学生画像与策略模板

- 对应需求：
  - `FR-01`
  - `FR-02`
- 目标：
  - 支持录入学生基础画像和默认防御型策略模板。
- 建议实现：
  - 新增学生信息表、策略配置表、默认初始化逻辑。
- 验收标准：
  - 新建学生后可读取其目标分数、白名单/黑名单模块。

### `ISSUE-M1-04` 题目输入与结构化解析

- 对应需求：
  - `FR-03`
  - `FR-04`
- 目标：
  - 支持文本和图片输入，并转成统一的 `ProblemInput`。
- 建议实现：
  - 增加图片路径、OCR 占位、信息充分性判断。
- 验收标准：
  - 输入不足时系统进入澄清模式。

### `ISSUE-M1-05` 错因归因引擎

- 对应需求：
  - `FR-05`
  - `FR-15`
- 目标：
  - 把问题归类为概念混淆、逻辑跳步或计算失误。
- 建议实现：
  - 从规则版起步，后续替换为真实模型。
- 验收标准：
  - 典型样例可稳定输出正确错因类型。

### `ISSUE-M1-06` 引导式讲解流程

- 对应需求：
  - `FR-06`
  - `FR-07`
- 目标：
  - 输出面向学生的单轮引导问题和视觉化提示。
- 建议实现：
  - 引入结构化响应对象；
  - 按模块输出视觉隐喻。
- 验收标准：
  - 默认不直接给完整答案；
  - 至少输出一条视觉提示。

### `ISSUE-M1-07` 黑白名单题型约束

- 对应需求：
  - `FR-08`
- 目标：
  - 在 M1 阶段先实现最小规则引擎。
- 建议实现：
  - 根据关键词或题型标签先判断是否超出防御范围。
- 验收标准：
  - 黑名单题型会触发限制提示，而非继续复杂展开。

### `ISSUE-M1-08` Gradio 本地界面

- 对应需求：
  - `FR-03`
  - `FR-06`
  - `NFR-04`
- 目标：
  - 建立可本地使用的最小 Web UI。
- 交付物：
  - `app/ui/gradio_app.py`
- 验收标准：
  - 用户可输入题目并看到学生视角输出。

### `ISSUE-M1-09` 错题本查询页

- 对应需求：
  - `FR-12`
- 目标：
  - 支持查看最近若干次问题记录。
- 建议实现：
  - 在 UI 增加“历史记录”页签。
- 验收标准：
  - 可按会话查看最近几轮记录。

### `ISSUE-M1-10` 会话状态对象与摘要

- 对应需求：
  - `FR-13`
  - `NFR-08`
- 目标：
  - 把多轮状态显式化，而不是只堆聊天记录。
- 建议实现：
  - 引入状态机、摘要字段和状态恢复逻辑。
- 验收标准：
  - 多轮后仍能保留当前教学目标和已排除路径。

### `ISSUE-M1-11` Prompt 模板版本化

- 对应需求：
  - `FR-15`
  - `NFR-03`
- 目标：
  - 将教学 Prompt、归因 Prompt、生成 Prompt 文件化管理。
- 验收标准：
  - Prompt 不再散落在业务逻辑代码中。

### `ISSUE-M1-12` 自动化测试底座

- 对应需求：
  - `NFR-08`
- 目标：
  - 建立单元测试和集成测试基本框架。
- 交付物：
  - `tests/unit`
  - `tests/integration`
- 验收标准：
  - 基础测试可运行并覆盖主链路骨架。
