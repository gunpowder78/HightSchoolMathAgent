# 学情评估与个性化补习计划智能工具规格说明

## 1. 文档目的

本文件面向 `HightSchoolMathAgent` 设计一套“学生高中数学学情评估 + 个性化补习计划”智能工具，并将两份研究报告中的核心结论转化为可开发、可测试、可追踪的规格资产。

该工具不是通用测评器，而是专门服务于：

- 广东新高考艺术生；
- 目标分数以 `90-100` 分保底区间为核心；
- 需要视觉降维、双轨重构、防御型提分和心理减压联动支持的学生群体。

## 2. 设计目标

工具必须同时完成四件事：

- `G1`：识别学生当前的分数底盘、错因结构、认知偏好和心理风险。
- `G2`：输出动态防御型提分策略，而不是静态押题清单。
- `G3`：围绕“主线 A 形与空间”与“主线 B 周期与旋转”生成个性化补习路径。
- `G4`：支持持续复评、计划调参和熔断修复，形成闭环。

## 3. 设计依据

### 3.1 来自研究报告的核心结论

- 新高考 `19` 题结构下，静态押宝特定模块存在坍塌风险。
- 艺术生在视觉空间加工上有优势，在纯代数链条上普遍脆弱。
- 防御型策略必须从“死守三个模块”升级为“基础问广覆盖 + 黑白名单约束 + 多选保守残缺得分”。
- 知识树需要拆成两条并行主线：
  - `主线 A`：平面几何直观 -> 向量 -> 解三角形 -> 空间向量立体几何；
  - `主线 B`：单位圆 -> 诱导公式 -> 三角函数图像 -> 整体换元与区间。
- 心理干预不能只有减负，还要建立掌控感、合法化放弃和多巴胺式微小成就反馈。

### 3.2 对现有项目基线的继承

本工具直接继承以下项目能力目标：

- `FR-01` 学生画像录入；
- `FR-02` 学习策略初始化；
- `FR-08` 白名单/黑名单题型约束；
- `FR-11` 练习反馈闭环；
- `FR-14` 家长视角报告。

## 4. 工具范围

### 4.1 首期范围

- 学生学情采集；
- 认知与风险画像计算；
- 防御型策略初始化；
- 个性化补习计划生成；
- 周度复评与计划调参；
- 家长/陪练者执行视图。

### 4.2 非范围

- 不直接替代完整辅导对话引擎；
- 不在首期做学校排名预测；
- 不做跨省份自适应录取模型；
- 不做复杂班级管理。

## 5. 角色

- `AR1 学生`
  - 提交基础信息、完成测评、接受训练计划。
- `AR2 家长/陪练者`
  - 配置时间、查看风险、执行每日训练。
- `AR3 系统`
  - 聚合数据、生成画像、输出计划、持续复评。

## 6. 核心概念

- `学情画像`
  - 对学生当前数学状态的结构化描述。
- `防御底盘`
  - 在当前条件下可稳定获取的基础分区能力。
- `动态防御策略`
  - 基于板块基础问覆盖、黑白名单与难度控制的策略集合。
- `主线 A`
  - 以视觉空间与向量代数统一为核心的补习主线。
- `主线 B`
  - 以单位圆和周期图像为核心的纯函数主线。
- `熔断修复`
  - 当学生连续在某一认知节点失效时，暂停推进并切换到降维修复脚本。

## 7. 功能需求

### 7.1 学情采集

- `APT-FR-01 基础档案采集`
  - 系统必须采集学生年级、选科、艺术方向、目标分数区间、每日可投入时长、当前学校环境类型。

- `APT-FR-02 数学经历采集`
  - 系统必须采集最近数学考试分数、常见失分模块、自我感受、最害怕题型、已放弃题型。

- `APT-FR-03 心理压力采集`
  - 系统必须采集学生在重点班压力、同侪比较焦虑、对数学的掌控感和回避行为信号。

### 7.2 学情评估

- `APT-FR-04 防御底盘测算`
  - 系统必须对学生在单选、多选、填空、解答题基础问上的可得分能力进行测算。

- `APT-FR-05 认知偏好识别`
  - 系统必须识别学生更偏向：
    - 视觉空间输入；
    - 机械步骤执行；
    - 图像化理解；
    - 纯符号推演耐受度。

- `APT-FR-06 模块能力矩阵`
  - 系统必须生成按模块拆分的能力矩阵，至少覆盖：
    - 平面向量；
    - 解三角形；
    - 立体几何/空间向量；
    - 三角函数；
    - 概率统计；
    - 解析几何基础问；
    - 导数基础问。

- `APT-FR-07 风险识别`
  - 系统必须识别以下风险：
    - 认知断崖风险；
    - 黑名单污染风险；
    - 习得性无助风险；
    - 学校/家庭策略冲突风险；
    - 时间预算不足风险。

### 7.3 策略与计划生成

- `APT-FR-08 动态防御策略生成`
  - 系统必须输出动态防御策略，而不是静态押题策略。
  - 输出必须覆盖：
    - 白名单模块；
    - 黑名单模块；
    - “所有大题第一问”是否纳入防区；
    - 多选保守残缺策略；
    - 解答题取分优先级。

- `APT-FR-09 双主线补习路径生成`
  - 系统必须根据学生画像生成 `主线 A` 与 `主线 B` 的补习路径。
  - 路径必须体现先后顺序、阶段目标和原子技能点。

- `APT-FR-10 日/周计划生成`
  - 系统必须输出：
    - 日计划；
    - 周计划；
    - 每次训练的输入素材、练习数量、验收方式和预期结果。

- `APT-FR-11 降维脚本推荐`
  - 对视觉型学生，系统必须优先推荐图像化、空间化和生活隐喻型脚本。
  - 对认知断崖节点，系统必须输出指定的熔断修复方案。

### 7.4 闭环与报告

- `APT-FR-12 三题切片复评`
  - 系统必须支持以“视觉复现题 + 机械执行题 + 费曼逆向讲解题”的微型测评进行阶段复评。

- `APT-FR-13 计划调参`
  - 当复评结果低于阈值时，系统必须自动建议：
    - 降速；
    - 重排主线顺序；
    - 切换脚手架；
    - 启动熔断修复。

- `APT-FR-14 家长执行面板`
  - 系统必须面向家长输出：
    - 今日重点；
    - 禁止训练项；
    - 鼓励话术；
    - 风险提醒；
    - 是否允许“合法化放弃”。

## 8. 非功能需求

- `APT-NFR-01 可解释性`
  - 所有评分、分层、推荐计划都必须说明依据。

- `APT-NFR-02 可调整性`
  - 计划不可写死，必须支持每周调参。

- `APT-NFR-03 低打击感`
  - 输出不得用“能力差”“完全不会”等破坏自尊的话术。

- `APT-NFR-04 可测试性`
  - 每个评分规则、分层逻辑、计划生成器都必须可单测。

- `APT-NFR-05 可追踪性`
  - 学情输入、评估结果、计划版本、调参原因都必须可追踪。

## 10. 评估模型设计

### 10.1 学情评分维度

- `D1 分数底盘`
  - 是否能稳住基础题。
- `D2 模块稳定度`
  - 某模块能否重复拿分。
- `D3 认知兼容度`
  - 当前学习路径是否符合学生认知偏好。
- `D4 心理承压度`
  - 学校压力是否已侵蚀学习掌控感。
- `D5 训练执行度`
  - 时间、完成度、复评表现是否稳定。

### 10.2 学生分层

- `L1 基础重建层`
  - 需要先解决掌控感与最低防御底盘。
- `L2 防御成形层`
  - 已有稳定基础，需要扩展防区广度。
- `L3 稳态提分层`
  - 已有一定得分闭环，需要提升波动控制和计划精细化。

## 11. 个性化计划生成规则

### 11.1 主线选择规则

- 若视觉空间优势明显且立几/向量基础较弱：
  - 主线 A 优先。
- 若三角函数混乱但图像感较好：
  - 主线 B 提前。
- 若学生对纯代数高度排斥：
  - 禁止以二次函数作为三角函数过渡桥梁。

### 11.2 风险驱动规则

- 若检测到重点班压力过高：
  - 计划必须输出“在校机械输出、回家降维输入”的执行建议。
- 若黑名单污染频发：
  - 计划必须缩小训练题源，强化白名单过滤。
- 若复评连续失败：
  - 自动进入熔断修复模式。

## 12. 关键工作流

### 12.1 首次建档工作流

1. 录入基础信息与目标。
2. 完成学情问卷与基础测评。
3. 计算分数底盘、模块矩阵、心理风险。
4. 选择或生成动态防御策略模板。
5. 生成首个 `2-4` 周补习计划。

### 12.2 周度复评工作流

1. 收集最近一周训练结果。
2. 执行三题切片复评。
3. 判断是否出现黑名单污染、连续失误或风险升高。
4. 调整主线顺序、题型边界、节奏与修复方案。
5. 输出新版本计划与家长说明。

## 13. 与现有系统架构的映射

建议新增以下组件：

- `Assessment Engine`
  - 负责画像计算、模块评分、风险识别。
- `Planning Engine`
  - 负责策略生成、日周计划编排、调参。
- `Policy Repository`
  - 存储白名单/黑名单规则、补习模板、熔断脚本。

在现有架构中的位置：

- 由 `Application 层` 编排；
- 读取 `Persistence 层` 中的学生、错题、诊断与练习记录；
- 由 `Model Adapter` 生成需要语言表达的计划说明；
- 与 `Tutor Engine`、`Practice Engine` 共用画像与状态对象。

## 15. 验收标准

- 工具必须能为一个学生生成可解释的学情画像。
- 工具必须能为不同认知风格学生生成不同计划，而不是套模板。
- 工具必须能识别至少三类风险并给出调参建议。
- 工具必须把“双主线 + 动态防御 + 熔断修复”体现在计划中。
- 所有结果必须可追溯到输入数据、规则或研究结论。
